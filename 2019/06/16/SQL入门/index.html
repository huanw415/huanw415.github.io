简介

SQL是什么

SQL: 结构化查询语言（Structured Query Language），是用于访问和处理数据库的标准语言，
也是一种ANSI（American National Standard Institute 美国国家标准组织）标准的计算机语言。
但是SQL仍然存在多种不同版本，大部分SQL数据库程序都有它们自己的专业扩展。 本文主要使用MySQL的语法进行示例。

SQL能做什么

SQL可用于

  1. 创建数据库
  2. 在数据库中创建表、创建存储过程、创建视图
  3. 在数据库中设置表、存储过程和视图的权限
  4. 面向数据库执行 增/删/改/查操作

排序：ORDER BY 关键字

可对结果集按照一列或多列进行升序或者降序排列。
默认按照升序ASC排列，需要进行降序排序时，可使用关键字DESC。

去重：DISTINCT 关键字

有时表中一列会有多个重复值，使用SELECT DISTINCT 可返回唯一不同值。

LIKE 模式搜索

查找column_name满足一个pattern的数据。

可以通过使用一些通配符来实现pattern，其中可用的通配符有一下集中
| 通配符 | 描述 |
| :————- | :————- |
| % | 替代0或多个字符 |
| _ | 替1个字符 |
|[charlist] | 匹配charlist里的任意字符 |
|[^charlist]或[!charlist] | 不匹配charlist里的任意字符 |
其中%和_均可和LIKE一起使用，[charlist]和[^charlist]和REGEXP正则一起搭配使用。
而且， LIKE和REGEXP均可以和NOT一起搭配使用，表示否定。

举个栗子：
WHERE name LIKE %oo%就是找到所有名字里有oo的数据

JOIN 多表

SQL join可将两个或者多个表的行结合起来。下图里是LEFT JOIN，RIGHT JOIN， INNER JOIN和OUT JOIN的其中用法。

INNER JOIN

A INNOER JOIN B当且仅当在B中找到和A匹配的数据时，该条数据才会被返回。

LEFT JOIN

A LEFT JOIN B：表示从左表A中返回所有的行，即使右表B中没有没有任何数据匹配。如果右表没有数据匹配则返回NULL。

LEFT JOIN table_b b ON a.key = b.key按照a表返回数据，即使在b表中找不到对应值，再使用b.key=NULL就可以滤除a中能在b表中找 到对应值的行，符合上图第二行左边第一个图的结果。

RIGHT JOIN

A RIGHT JOIN B：表示从右表B中返回所有的行，即使左表A中没有没有任何数据匹配。如果左表没有数据匹配则返回NULL。

OUTER JOIN

A OUTER JOIN B: 表示只要左表A或者右表B中任意一个表中存在，则返回该行数据。

SQL 约束（constraints）

用于规定表中的数据规则。如果存在违反约束的数据行为，行为会被约束终止。
在SQL中有的约束如下：

  1. NOT NULL: 不能存储为NULL的值
  2. UNIQUE: 保证某列的每行必须有惟一值
  3. PRIMARY KEY: 是NOT NULL 和UNIQUE的结合。确保某列（或两个列多个列的结合）有唯一标识，有助于快速检索某个特定记录
  4. FOREIGN KEY: 保证一个表中的数据和另一个表中的值得参照完整性
  5. CHECK：保证列中的值符合指定条件
  6. DEFAULT：规定没有给列赋值时的默认值

注意：每个表可以有多个UNIQUE约束，但是每个表只能有一个PRIMARY KEY约束

PRIMARY KEY

PRIMARY KEY唯一标识数据库的每条记录。每个表都应该有一个主键，并且有且仅有一个，且不能为NULL。
